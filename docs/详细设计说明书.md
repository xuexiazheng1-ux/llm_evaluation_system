### 2.2 技术栈
- **后端开发语言**：Python 3.11+ (FastAPI)
- **任务队列**：Celery + Redis
- **数据库**：PostgreSQL 15 (元数据)，Elasticsearch 8 (日志)，Prometheus (指标)，Jaeger (链路)
- **对象存储**：MinIO
- **消息队列**：Apache Kafka (用于异步事件)
- **容器化**：Docker + Kubernetes (部署)
- **前端**：React 18 + TypeScript + Ant Design
- **AI辅助**：LangChain + 本地LLM (如Qwen-14B) 或 OpenAI API
- **测试框架**：DeepEval, Playwright, PyTest, Locust, JMeter, Chaos Mesh

## 3. 模块详细设计

### 3.1 测试编排中心
#### 3.1.1 功能描述
- 接收用户创建的测试任务（单次/周期/门禁触发）
- 解析任务配置（包含测试类型、评测集、环境参数、门禁阈值）
- 调度各执行模块，协调工作流（如先跑安全测试，再跑自动化，最后评测）
- 收集各模块结果，判断是否通过门禁
- 与CI/CD系统集成（通过Webhook或API）

#### 3.1.2 关键数据结构
```python
# 任务定义
class TestTask:
    id: str
    name: str
    type: enum('manual', 'periodic', 'gate')
    status: enum('pending', 'running', 'passed', 'failed', 'error')
    config: dict  # 包含测试范围、环境、阈值等
    created_by: str
    created_at: datetime
    started_at: datetime
    finished_at: datetime
    result_summary: dict  # 通过率、失败项等

# 任务步骤
class TaskStep:
    id: str
    task_id: str
    step_type: enum('eval', 'api_test', 'e2e_test', 'scene_sim', 'security_test')
    status: enum('pending', 'running', 'success', 'failed')
    result_ref: str  # 结果存储位置（MinIO路径）
    retry_count: int
    started_at: datetime
    finished_at: datetime
```

### 3.2 评测集管理服务
#### 3.2.1 核心逻辑
- 评测集存储：每个评测集包含多个测试用例，支持版本化（Git-like 版本管理）。
- 评分规则：内置DeepEval指标，支持自定义GEval（自然语言描述）。
- 漂移监控：定时任务对比线上版本与基线版本得分，超出阈值发送告警。

#### 3.2.2 数据库表设计
```sql
-- 评测集表
CREATE TABLE eval_dataset (
    id UUID PRIMARY KEY,
    name VARCHAR(255),
    description TEXT,
    version INT,
    parent_version INT, -- 上一个版本
    created_by VARCHAR(100),
    created_at TIMESTAMP,
    tags JSONB
);

-- 测试用例表
CREATE TABLE test_case (
    id UUID PRIMARY KEY,
    dataset_id UUID REFERENCES eval_dataset(id),
    input TEXT, -- 用户输入/对话历史
    expected_output TEXT, -- 期望输出（可选）
    context TEXT, -- 上下文信息（用于RAG）
    tools JSONB, -- 可用工具定义
    metadata JSONB, -- 标签、难度等
    created_at TIMESTAMP
);

-- 评分规则表
CREATE TABLE scoring_rule (
    id UUID PRIMARY KEY,
    name VARCHAR(255),
    rule_type ENUM('predefined', 'geval', 'dag'),
    metric_name VARCHAR(100), -- 如 answer_relevancy
    config JSONB, -- GEval规则描述/DAG定义
    created_at TIMESTAMP
);

-- 漂移监控任务表
CREATE TABLE drift_monitor (
    id UUID PRIMARY KEY,
    dataset_id UUID REFERENCES eval_dataset(id),
    baseline_version INT,
    frequency ENUM('hourly', 'daily', 'weekly'),
    threshold DECIMAL(5,2), -- 百分比
    last_run TIMESTAMP,
    next_run TIMESTAMP,
    alert_channels JSONB -- 邮件/钉钉等
);
```

#### 3.2.3 DeepEval集成
- 评测执行时，服务调用DeepEval的`evaluate()`方法，传入测试用例列表和评分规则。
- 自定义GEval通过DeepEval的`GEval`类实现，配置参数从`config`字段解析。
- DAG指标通过DeepEval的`DAG`类构建决策树，同样从`config`解析节点定义。
- 评测结果（TestResult）转换为内部格式存储到MinIO，并更新任务步骤状态。

| 端点 | 方法 | 功能 |

| `/api/v1/datasets` | GET/POST | 列表/创建评测集 |
| `/api/v1/datasets/{id}/versions` | GET | 获取版本列表 |
| `/api/v1/datasets/{id}/versions/{version}` | GET | 获取特定版本用例 |
| `/api/v1/datasets/{id}/testcases` | POST | 批量导入用例 |

### 3.3 自动化执行服务
#### 3.3.1 模块组成
- **API测试引擎**：基于PyTest，支持OpenAPI schema校验，测试脚本模板化。
- **E2E测试引擎**：基于Playwright，支持录制回放，提供可视化调试。
- **环境管理器**：通过Kubernetes Job动态创建/销毁测试环境，支持私有化环境配置注入。
- **测试数据生成器**：集成AI辅助，根据自然语言描述生成测试数据。

#### 3.3.2 关键技术实现
- **API测试脚本生成**：用户提供OpenAPI文档和自然语言描述，AI辅助服务生成PyTest代码，存入MinIO，执行时拉取运行。
- **Playwright脚本录制**：提供Web界面启动录制，用户操作后生成脚本，可保存到项目中。
- **环境配置**：测试环境配置以Kubernetes YAML模板形式存储，执行时渲染变量（如镜像版本、环境变量）。

#### 3.3.3 数据模型
```sql

CREATE TABLE test_script (

```

### 3.4 场景模拟服务
#### 3.4.1 模拟能力
- **多轮对话模拟**：基于状态机，每个状态定义可能的用户输入和预期Agent响应。使用开源库如`dialogue-kit`或自研。
- **工具链路模拟**：集成MockServer，根据工具调用请求返回预定义的Mock数据。
- **异步任务模拟**：通过Celery任务模拟长耗时操作，并记录状态变化。
- **并发模拟**：调用Locust/JMeter分布式运行，从Kubernetes Job启动压测。
- **混沌实验**：通过Chaos Mesh注入故障（网络延迟、Pod重启、CPU耗尽），与可观测性服务联动。

#### 3.4.2 数据模型
```sql
-- 对话剧本表
CREATE TABLE dialogue_script (
    id UUID PRIMARY KEY,
    name VARCHAR(255),
    states JSONB, -- 状态机定义

);

-- 工具Mock规则表
CREATE TABLE tool_mock (
    id UUID PRIMARY KEY,
    tool_name VARCHAR(255),
    request_pattern JSONB, -- 匹配条件
    response_data JSONB,   -- 返回内容
    delay INT,             -- 模拟延迟（ms）
    error_rate DECIMAL(3,2) -- 错误率
);
```

### 3.5 可观测性服务
#### 3.5.1 数据采集
- **Tracing**：通过OpenTelemetry SDK自动埋点（FastAPI中间件、DeepEval集成、Playwright测试等），导出至Jaeger。
- **Logs**：结构化日志（JSON格式），通过Filebeat采集至Elasticsearch。
- **Metrics**：业务指标（测试执行次数、通过率、耗时）通过Prometheus客户端暴露，由Prometheus抓取。

#### 3.5.2 根因分析
- **失败分类**：从Trace中提取错误节点，根据错误类型（超时、权限、模型拒绝）分类，存储到Elasticsearch。
- **AI辅助分析**：调用AI辅助服务，传入Trace和错误日志，返回根因可能性排序和修复建议。
- **复现脚本生成**：根据Trace中的请求序列，自动生成Python脚本（使用requests库）或Playwright脚本，存入MinIO并提供下载。

#### 3.5.3 API设计
| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/v1/traces/{trace_id}` | GET | 获取Trace详情 |
| `/api/v1/analysis/failure/{test_result_id}` | POST | 触发根因分析 |
| `/api/v1/analysis/reproduce/{trace_id}` | POST | 生成复现脚本 |

### 3.6 安全测试服务
#### 3.6.1 测试项实现
- **Prompt Injection**：维护攻击向量库（JSON文件），对Agent发送攻击提示，检测是否执行恶意指令或泄露敏感信息。
- **越权测试**：模拟不同角色用户（通过认证头），尝试调用高权限工具。
- **RAG权限绕过**：构造查询试图访问未授权文档，验证返回结果是否包含。
- **敏感信息扫描**：正则匹配+AI模型检测回复中是否包含身份证、手机号等。
- **审计验证**：查询审计日志表，检查关键操作是否有记录。

#### 3.6.2 数据模型
```sql
-- 攻击向量库
CREATE TABLE attack_vector (
    id UUID PRIMARY KEY,
    type ENUM('prompt_injection', 'jailbreak', 'sql_injection'),
    content TEXT,
    expected_result VARCHAR(255) -- 预期防御结果
);

-- 安全测试任务
CREATE TABLE security_test_task (
    id UUID PRIMARY KEY,
    target_agent_id VARCHAR(255),
    vectors_used JSONB, -- 使用的攻击向量ID列表
    result JSONB,       -- 检测结果
    executed_at TIMESTAMP
);
```

### 3.7 报告服务
#### 3.7.1 报告生成
- **模板化**：使用Jinja2模板定义HTML/PDF报告格式。
- **AI总结**：调用AI辅助服务，将测试数据转化为自然语言总结。
- **自动录入缺陷**：根据失败用例，通过Jira API创建缺陷，关联Trace ID。

#### 3.7.2 质量仪表盘
- 基于Grafana + Prometheus + Elasticsearch数据源，展示关键指标。
- 提供自定义面板，支持按项目、版本、模块筛选。

### 3.8 AI辅助服务
#### 3.8.1 功能列表
- **测试脚本生成**：输入需求描述，输出PyTest/Playwright代码。
- **对抗样本生成**：基于现有测试用例，生成语义变体。
- **日志分析辅助**：输入日志和Trace，输出可能的错误原因。
- **文档生成**：根据测试结果生成测试计划、缺陷分析报告。

#### 3.8.2 技术实现
- 使用LangChain构建链式调用，模型可选择本地部署的LLM（如Qwen-14B）或云端API。
- 提示词工程优化，保证输出结构化。
- 结果后处理：代码格式化、语法检查。

## 6. 关键技术实现细节
### 6.1 DeepEval集成
- 封装DeepEval调用为独立服务，通过gRPC或HTTP API提供给评测集管理服务。
- 自定义指标示例（DAG）：
```python
from deepeval.metrics import DAG
from deepeval.test_case import LLMTestCase

dag = DAG(
    name="结构化摘要评测",
    nodes=[
        {"name": "包含关键点", "type": "GEval", "criteria": "摘要是否包含...", "weight": 0.5},
        {"name": "顺序正确", "type": "GEval", "criteria": "关键点顺序是否...", "weight": 0.3},
        {"name": "长度合规", "type": "GEval", "criteria": "摘要长度是否在...", "weight": 0.2}
    ]
)
result = dag.measure(test_case)
```

### 6.2 异步任务处理
- 使用Celery + Redis作为任务队列，每个测试步骤作为一个任务。
- 编排中心协调任务依赖（使用Canvas的chain/group）。
- 任务状态实时更新到PostgreSQL，WebSocket推送前端。

### 6.3 可观测性集成
- FastAPI应用中添加OpenTelemetry中间件：
```python
from opentelemetry.instrumentation.fastapi import FastAPIInstrumentor
FastAPIInstrumentor.instrument_app(app)
```
- 自定义Span记录关键操作（如DeepEval评测、测试执行）。
- Trace ID透传：通过HTTP头`traceparent`在微服务间传递。

### 6.4 AI辅助服务实现
- 提供统一接口`/api/v1/ai/generate`，接收`prompt`和`context`，返回生成内容。
- 使用LangChain的`LLMChain`，针对不同任务（代码生成、日志分析）设计不同提示模板。
- 支持流式输出（Server-Sent Events）用于前端实时显示生成过程。

## 7. 部署架构
### 7.1 Kubernetes部署
- 每个微服务部署为独立的Deployment，配置HPA自动伸缩。
- 使用Ingress暴露API网关。
- 持久化：PostgreSQL使用StatefulSet，MinIO使用分布式模式。
- 监控：Prometheus Operator + Grafana + Jaeger Operator。

### 7.2 私有化部署
- 提供Helm Chart，一键部署到客户K8s集群。
- 支持离线安装（所有依赖镜像打包）。
- 配置项包括：数据库连接、存储类、LLM模型地址（本地或云端）。

## 8. 安全设计
- **认证与授权**：JWT + RBAC，API网关层验证Token，服务间使用mTLS。
- **数据加密**：敏感数据（如数据库连接串、API密钥）使用Kubernetes Secret存储，并启用加密。
- **审计日志**：所有修改操作记录到`audit_log`表，包括操作人、时间、IP、操作内容。
- **漏洞扫描**：定期使用Trivy扫描镜像，确保无高危漏洞。

## 9. 开发与测试
### 9.1 开发环境
- 本地使用Docker Compose启动所有依赖（PostgreSQL、Redis、MinIO等）。
- 微服务可在IDE中运行，通过环境变量连接依赖。
- API文档自动生成（FastAPI Swagger UI）。

### 9.2 测试策略
- **单元测试**：每个微服务独立测试，覆盖率目标80%。
- **集成测试**：使用Testcontainers启动真实依赖，测试跨服务流程。
- **端到端测试**：部署到测试K8s环境，运行完整测试场景。
- **性能测试**：使用Locust模拟高并发，监控系统指标。
