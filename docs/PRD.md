# AI测试与评测系统产品需求文档（PRD）

## 1. 产品背景与目标

### 1.1 背景
随着B端Agent（智能体）在复杂业务场景中的广泛应用，其质量保障面临前所未有的挑战。传统测试工具无法应对LLM输出的不确定性、多轮对话的复杂性、工具链路的多样性和安全合规的严苛要求。现有开源工具（如DeepEval）虽在LLM评测方面表现优异，但在自动化测试执行、环境管理、压力测试、可观测性及根因分析等方面存在明显短板。

为满足AI测试工程师岗位（见附件）的核心技能要求，需构建一套**企业级AI测试与评测系统**，覆盖从离线评测、自动化测试到线上监控的全流程，确保Agent的稳定性、安全性、可观测性和可复现性。

### 1.2 目标
- **建立统一的质量门禁**：集成CI/CD，自动拦截不符合质量指标的版本。
- **支持复杂Agent场景测试**：模拟多轮对话、工具调用、异步任务、并发与异常恢复等。
- **实现评测自动化与智能化**：利用DeepEval等工具进行LLM-as-judge评分，结合AI辅助生成测试用例和报告。
- **提供可观测性与调试能力**：基于Tracing/Metrics/Logs实现失败分类、根因分析和自动复现脚本生成。
- **确保安全合规**：覆盖Prompt Injection、越权调用、敏感信息泄露等Agent专属安全测试。

## 2. 用户场景与角色

### 2.1 核心用户
- **AI测试工程师**：负责编写测试用例、执行测试、分析缺陷、维护质量体系。
- **开发工程师**：自测阶段使用系统快速验证代码质量，定位问题。
- **运维/可靠性工程师**：监控线上质量，触发漂移告警，参与混沌工程实验。
- **测试经理/质量负责人**：查看质量报告，制定质量标准，管理测试资产。

### 2.2 典型场景
- **版本发布前**：测试工程师配置质量门禁，系统自动运行全量测试（评测+自动化+安全），生成报告，拦截不达标版本。
- **新场景探索**：测试工程师利用AI辅助生成对抗样本，扩充评测集。
- **线上问题复现**：通过Trace自动生成复现脚本，快速定位根因。
- **私有化部署验证**：系统提供针对客户环境的测试清单，一键执行合规性检查。

## 3. 功能需求

### 3.1 评测集管理模块（基于DeepEval增强）
- **测试用例管理**：
  - 支持导入/导出评测集（JSON格式）。
  - 支持批量导入测试用例。
  - 支持单条测试用例的增删改查。
  - 测试用例字段：input（输入）、expected_output（期望输出）、context（上下文）、metadata（元数据）。
- **评分规则定义**：
  - 内置DeepEval指标：Answer Relevancy、Faithfulness、Contextual Relevancy。
  - 支持自定义GEval规则（自然语言描述评分标准）。
  - 支持多LLM提供商进行评分（OpenAI、DeepSeek等）。
- **任务管理**：
  - 支持任务历史记录与重跑功能。
  - 支持查看详细评测结果（得分、评估说明、优化建议）。
  - 支持配置待评测系统API地址（如AnythingLLM）。

### 3.2 自动化测试执行引擎
- **API测试**：
  - 基于PyTest框架，支持OpenAPI Schema校验、工具调用链路验证。
  - 自动生成接口测试用例（利用Cursor/Claude Code辅助）。
- **E2E测试**：
  - 集成**Playwright**，支持React管理台/编排器的UI自动化。
  - 录制回放功能，降低脚本编写成本。
- **环境管理**：
  - 支持SaaS环境动态创建/销毁（通过Kubernetes Job）。
  - 支持私有化环境配置注入（如域名、密钥、脱敏开关）。
- **测试数据生成**：
  - 集成AI代码生成插件，自动补充边界场景和异常数据。
  - 支持从生产环境脱敏抽取真实数据（需合规审批）。

### 3.3 Agent场景模拟器
- **多轮对话模拟**：
  - 基于状态机定义对话剧本，支持随机路径和条件分支。
  - 集成**对话模拟库**（如常见客服、销售场景）。
- **工具链路模拟**：
  - 模拟Agent调用外部工具（数据库、API、内部服务）的返回，支持超时、错误、异常响应。
  - 集成MockServer，动态配置Mock规则。
- **异步/长任务模拟**：
  - 模拟长时间运行的任务，验证状态轮询、超时处理和回调机制。
- **并发与异常恢复**：
  - 结合**Locust/JMeter**进行压力测试，模拟多用户并发。
  - 集成**Chaos Mesh**注入故障（网络延迟、服务重启、资源耗尽）。
- **幂等与重试风暴测试**：
  - 自动构造重复请求，验证幂等性设计。
  - 模拟大量重试请求，检测系统保护机制。

### 3.4 可观测性与调试模块
- **数据采集**：
  - 集成**OpenTelemetry**，自动采集Tracing（包括LLM调用、工具调用）、Logs、Metrics。
  - 支持主流数据后端（Jaeger、Elasticsearch、Prometheus）。
- **失败分类与根因分析**：
  - 利用AI分析Trace和日志，自动分类失败原因（如工具超时、模型拒绝、权限不足）。
  - 生成**根因分析报告**，指出可能的代码位置或配置错误。
- **复现脚本生成**：
  - 根据Trace自动生成可执行的复现脚本（Python、Curl、Playwright代码）。
  - 支持一键在测试环境重放请求。

### 3.5 安全合规测试模块
- **Prompt Injection检测**：
  - 集成专用攻击向量库（如提示词注入、越狱尝试），自动执行并验证结果。
- **越权工具调用测试**：
  - 模拟低权限用户尝试调用高权限工具，检测权限校验缺失。
- **RAG权限绕过**：
  - 构造查询试图访问未授权文档，验证RAG检索的权限控制。
- **敏感信息泄露扫描**：
  - 检测回复中是否包含密钥、身份证号、手机号等敏感信息。
- **审计链路完整性验证**：
  - 检查操作日志是否完整记录（谁、何时、调用了什么工具、参数）。
- **离线数据脱敏校验**：
  - 针对私有化部署，验证数据脱敏规则是否生效。

### 3.6 报告与闭环管理
- **智能报告生成**：
  - 利用AI（Claude/GPT）总结测试结果，生成人类可读的测试计划、缺陷分析报告、复盘总结。
  - 提供**私有化部署测试清单**，按环境生成检查项。
- **缺陷自动录入**：
  - 支持与Jira、工单系统对接，自动创建缺陷，关联Trace和复现步骤。
- **质量仪表盘**：
  - 展示质量趋势、各模块通过率、漂移监控曲线。
  - 支持自定义报表导出（PDF/Excel）。

### 3.7 AI辅助能力（贯穿所有模块）
- **测试脚本生成**：输入自然语言描述，自动生成PyTest/Playwright代码。
- **对抗样本生成**：基于现有评测集，利用AI生成变体对抗样本。
- **日志分析辅助**：在根因分析界面提供AI建议，指出常见模式。
- **文档自动生成**：根据测试执行结果，自动生成测试用例文档和检查清单。

## 4. 非功能需求

### 4.1 性能
- 评测执行：单个测试用例平均耗时<5秒（依赖LLM调用时间）。
- 并发测试：支持100+并发虚拟用户。
- 数据存储：Tracing数据保留30天，支持冷热分离。

### 4.2 可扩展性
- 模块化设计，各组件可独立升级/替换。
- 支持插件化扩展（如新增评测指标、测试框架、数据源）。

### 4.3 安全性
- 所有敏感数据（密钥、Token）加密存储。
- 私有化部署支持完全离线运行，不依赖外部服务。
- 严格的权限控制：RBAC模型，区分管理员、测试工程师、只读用户。

### 4.4 易用性
- 提供Web操作界面，支持可视化配置测试任务、查看报告。
- 提供命令行工具（CLI），便于集成CI/CD。
- 提供详细的API文档和SDK（Python/Java）。

### 5.2 技术栈（MVP已实现）
- **后端**：Python (FastAPI) + SQLAlchemy 2.0 + Celery（异步任务）
- **前端**：React + Ant Design
- **评测核心**：DeepEval（集成自定义指标）
- **数据存储**：PostgreSQL（主数据库）、Redis（Celery队列）
- **部署**：Docker Compose

### 5.3 技术栈（未来规划）
- **自动化测试**：Playwright、PyTest、Locust、JMeter
- **混沌工程**：Chaos Mesh
- **可观测性**：OpenTelemetry Collector, Jaeger, Prometheus, Elasticsearch
- **AI辅助**：LangChain + 本地LLM（如Llama 3）或云端API
- **对象存储**：MinIO（测试报告）

### 5.4 集成DeepEval的具体方式
- DeepEval作为评测模块的基础库，通过Python API调用。
- 使用DeepEval预定义指标：AnswerRelevancyMetric、FaithfulnessMetric、ContextualRelevancyMetric。
- 支持自定义GEval指标，通过自然语言描述评分标准。
- 针对DeepSeek等国产模型，实现自定义Evaluator绕过DeepEval模型限制。
- 评测结果存入PostgreSQL数据库，通过Web界面展示。
- 评测前自动清洗`<think>`标签内容，确保评分准确性。

## 6. 产品Roadmap

### 阶段一（1-2个月）：MVP（已完成）
- 实现评测集管理、测试用例CRUD、JSON导入导出。
- 集成DeepEval核心指标（Answer Relevancy、Faithfulness、Contextual Relevancy、GEval）。
- 支持自定义GEval评分规则。
- 实现同步/异步评测执行，Celery任务队列。
- 支持任务历史记录与重跑功能。
- 提供详细评测报告（得分、评估说明、优化建议）。
- 支持多LLM提供商（OpenAI、DeepSeek）进行评分。
- 支持配置待评测系统API（如AnythingLLM）。
- 支持CI/CD集成，提供质量门禁API。
- 提供Web界面（React + Ant Design）和CLI工具。

### 阶段二（2-3个月）：自动化增强
- 集成Playwright和PyTest，实现E2E和API测试。
- 开发环境管理模块（Kubernetes Job）。
- 添加基础场景模拟（多轮对话、工具链路）。

### 阶段三（2-3个月）：可观测性与安全
- 集成OpenTelemetry，实现Tracing采集和复现脚本生成。
- 添加根因分析AI辅助。
- 实现安全合规测试模块（Prompt Injection、越权等）。

### 阶段四（持续）：优化与扩展
- 支持更多评测指标和框架。
- 增强AI辅助能力（自动生成测试用例）。
- 提供SaaS版本和多租户支持。

## 7. 成功指标
- **覆盖率**：覆盖岗位要求的7大核心技术技能。
- **效率提升**：测试脚本编写时间减少50%，缺陷定位时间减少70%。
- **质量拦截**：发布前拦截至少90%的严重缺陷。
- **用户满意度**：测试工程师NPS ≥ 40。
