# AI测试与评测系统 - 前端交互逻辑设计

## 1. 引言
本文档旨在定义AI测试与评测系统前端的交互逻辑，为UI/UX设计师和前端开发提供明确指引。系统面向测试工程师、开发人员、测试经理等角色，提供可视化的工作台，以高效完成Agent质量保障工作。

## 2. 用户角色与核心任务
| 角色 | 核心任务 |
|------|----------|
| **测试工程师** | 创建/执行测试任务、管理评测集、编写自动化脚本、配置场景模拟、分析缺陷 |
| **开发工程师** | 自测代码、查看测试报告、定位失败原因 |
| **测试经理** | 查看质量仪表盘、配置门禁策略、分析质量趋势 |
| **运维人员** | 管理测试环境、查看可观测性数据 |

## 3. 全局交互约定
- **导航结构**：左侧固定侧边栏，顶部为用户信息/通知/快捷操作。
- **主题**：支持亮色/暗色模式切换。
- **反馈机制**：操作后显示Toast提示（成功/错误），重要状态变更（如任务完成）通过通知中心推送。
- **实时更新**：任务执行状态、仪表盘数据通过WebSocket实时推送更新。
- **响应式**：优先桌面端设计，但适配平板（1366×768以上分辨率）。

## 4. 核心页面设计

### 4.1 登录页
- **功能**：用户登录、SSO集成。
- **交互**：输入账号密码/第三方认证，登录后跳转仪表盘。

### 4.2 仪表盘
- **布局**：顶部为关键指标卡片（今日执行任务数、通过率、漂移告警数），下方为趋势图表（质量趋势、任务耗时），右侧为待办任务列表。
- **交互**：
  - 点击卡片跳转到对应列表页。
  - 趋势图支持时间范围筛选（近7天/30天），鼠标悬停显示具体数据。
  - 待办任务可直接点击跳转到任务详情。

### 4.3 任务管理

#### 4.3.1 任务列表页
- **功能**：查看所有测试任务，支持搜索、筛选（状态、类型、创建人）、排序。
- **交互**：
  - 列表展示任务名称、状态、创建时间、执行人、操作（查看报告、重新运行）。
  - 点击任务名称进入任务详情页。
  - 支持批量操作（删除、停止）。
  - 右上角“新建任务”按钮，打开任务创建向导。

#### 4.3.2 任务创建向导（多步骤表单）
- **步骤1：基本信息**：任务名称、描述、类型（手动/周期/门禁）、触发方式（立即/定时）。
- **步骤2：测试范围**：
  - 评测集：多选并指定版本。
  - 自动化脚本：多选API/E2E脚本。
  - 场景模拟：选择对话剧本、工具Mock规则。
  - 安全测试：开关及选择攻击向量。
- **步骤3：环境配置**：
  - 环境类型（SaaS/私有化），选择模板，填写变量（如镜像标签）。
- **步骤4：门禁阈值**（仅门禁类型）：各测试项通过率阈值设置。
- **步骤5：确认并提交**：展示概览，点击“立即执行”或“保存为模板”。
- **交互细节**：
  - 每一步均有校验，下一步按钮禁用直至必填项完成。
  - 评测集选择时提供版本对比预览。
  - 环境变量支持动态表单（键值对输入）。

#### 4.3.3 任务详情页
- **布局**：上方为任务元信息（状态、时间、配置摘要），中间为执行步骤列表（以时间线展示），下方为报告预览/下载区域。
- **交互**：
  - 步骤列表可展开查看详细日志和结果。
  - 失败步骤旁边有“分析”按钮，点击调用根因分析，弹出侧滑面板展示分析结果和复现脚本下载链接。
  - 实时状态更新：步骤进度条、状态图标随WebSocket推送变化。
  - 支持“停止任务”、“重新运行”操作。

### 4.4 评测集管理

#### 4.4.1 评测集列表页
- **功能**：展示所有评测集，支持搜索、标签筛选。
- **交互**：
  - 卡片式或列表式展示，卡片上显示名称、版本数、最后更新时间。
  - 点击卡片进入评测集详情页。
  - 右上角“新建评测集”按钮，进入创建页。

#### 4.4.2 评测集详情页
- **布局**：
  - 左侧为版本列表树（类似Git分支）。
  - 右侧上方为基本信息（名称、描述、标签）。
  - 右侧中间为测试用例列表（分页展示）。
  - 右侧下方为评分规则列表。
- **交互**：
  - 切换版本：点击左侧版本，右侧用例列表更新为该版本内容。
  - 版本对比：选择两个版本，右侧以diff形式展示用例差异。
  - 用例管理：支持增删改查，批量导入（拖拽上传JSON/CSV）。
  - 评分规则：支持添加预定义指标（下拉选择）或自定义GEval（弹出编辑器，自然语言输入）。
  - 漂移监控：在版本树旁边有“设置漂移监控”按钮，打开配置弹窗（频率、阈值、告警渠道）。
  - 右上角“执行评测”按钮，快速对该评测集发起评测任务（跳转任务创建向导并预填评测集）。

#### 4.4.3 自定义GEval编辑器
- **功能**：用自然语言定义评分标准，支持多步骤、加权。
- **交互**：
  - 提供模板示例。
  - 支持添加多个评分标准，每个标准可设置权重。
  - 实时预览：输入测试用例，模拟评分结果。
  - 保存后自动转换为GEval配置。

### 4.5 自动化脚本管理

#### 4.5.1 脚本列表页
- **功能**：展示API/E2E脚本，支持类型筛选、搜索。
- **交互**：
  - 列表展示脚本名称、类型、框架、最后修改人、操作（编辑、运行、复制）。
  - 点击“运行”快速执行脚本（弹出环境选择框）。
  - 右上角“新建脚本”按钮，进入脚本编辑器。

#### 4.5.2 脚本编辑器
- **功能**：在线编写/调试测试脚本。
- **交互**：
  - 代码编辑器（Monaco Editor）支持语法高亮、自动补全（Python/JavaScript）。
  - 右侧工具栏：保存、格式化、运行调试、AI辅助生成。
  - AI辅助生成：点击“AI生成”按钮，弹出侧边栏输入需求描述，生成后插入光标位置。
  - 调试模式：选择环境后运行，实时输出日志，可设置断点（通过Playwright Inspector集成）。

#### 4.5.3 录制回放（仅E2E）
- **功能**：通过Playwright录制用户操作生成脚本。
- **交互**：
  - 点击“开始录制”，弹出新窗口加载目标URL，用户操作，录制工具条显示录制状态。
  - 停止录制后，自动生成Playwright代码并填充到编辑器。

### 4.6 场景模拟配置

#### 4.6.1 对话剧本管理
- **功能**：创建/编辑多轮对话剧本。
- **交互**：
  - 可视化状态机编辑器：拖拽添加状态，连线定义转移，双击状态编辑内容（用户输入、预期Agent响应）。
  - 支持导入/导出JSON定义。
  - 提供模拟测试功能：点击“运行模拟”，展示对话过程。

#### 4.6.2 工具Mock规则管理
- **功能**：配置Mock规则，模拟外部工具调用。
- **交互**：
  - 规则列表展示工具名称、请求匹配模式、响应数据。
  - 新建规则：表单输入工具名、请求匹配条件（JSONPath），响应数据（支持静态JSON或动态脚本）。
  - 支持导入Postman Collection/Mockoon数据。

### 4.7 安全测试配置
- **功能**：管理攻击向量库、执行安全测试。
- **交互**：
  - 攻击向量库列表（类型、内容、预期防御结果），支持增删改、批量导入。
  - 快速执行安全测试：选择目标Agent和向量，一键运行，展示检测结果（通过/失败/告警）。

### 4.8 报告与仪表盘

#### 4.8.1 报告列表
- **功能**：查看所有生成的报告（按任务关联）。
- **交互**：列表展示报告名称、任务名称、生成时间、操作（预览、下载PDF/HTML）。

#### 4.8.2 报告预览页
- **功能**：在线查看测试报告。
- **交互**：
  - 左侧为报告大纲（章节导航），右侧为内容。
  - 包含AI总结摘要，以及详细测试数据表格、失败用例列表。
  - 失败用例可点击查看Trace详情和复现脚本。
  - 导出按钮支持多种格式。

#### 4.8.3 质量仪表盘（Grafana嵌入）
- **功能**：展示质量趋势、漂移监控、通过率等。
- **交互**：
  - 嵌入式Grafana面板，保留Grafana的交互（时间选择、钻取）。
  - 可自定义仪表盘布局（保存视图）。

### 4.9 设置与用户管理
- **功能**：用户管理（管理员）、个人资料、API密钥、系统配置。
- **交互**：
  - 用户列表（增删改、分配角色）。
  - API密钥生成/吊销。
  - 系统配置（LLM模型选择、告警邮箱、CI/CD集成Webhook）。

## 5. 关键交互流程示例

### 5.1 场景：测试工程师执行一次版本门禁测试
1. 登录后进入仪表盘，点击“新建任务”。
2. 在向导中：
   - 任务名称：release-v1.2.3-gate
   - 类型：门禁
   - 测试范围：勾选评测集（核心集v3）、自动化脚本（API基础测试、E2E主流程）、场景模拟（对话压力测试）、安全测试（全开）
   - 环境：选择“生产镜像环境”，填写镜像标签v1.2.3
   - 门禁阈值：评测通过率≥95%，API通过率100%，安全高危漏洞0
3. 点击提交，任务进入pending状态，跳转任务详情页。
4. 任务开始执行，步骤列表实时更新，可看到安全测试先完成，绿色通过；评测进行中。
5. 评测步骤失败，通过率92%低于阈值。步骤状态变红，旁边出现“分析”按钮。
6. 点击“分析”，侧滑面板显示根因分析结果：“主要失败原因是意图分类错误，集中在对抗样本#23、#45”，并提供复现脚本下载。
7. 测试工程师下载脚本，本地复现确认问题，提交缺陷至Jira。
8. 任务最终状态为failed，门禁拦截成功。

### 5.2 场景：测试经理查看质量趋势
1. 点击左侧导航“质量仪表盘”。
2. 查看近30天评测通过率趋势图，发现上周有明显下降。
3. 鼠标悬停某天，显示当天失败用例最多的评测集。
4. 点击钻取，跳转到该评测集详情页，并自动筛选出当天失败的用例，便于进一步分析。

## 6. 可视化设计要点
- **颜色语义**：绿色代表成功，红色代表失败，黄色代表进行中/警告，灰色代表未开始/跳过。
- **图表类型**：趋势图使用折线图；通过率分布使用饼图/柱状图；失败分类使用饼图/树图。
- **字体与间距**：遵循Ant Design规范，保证可读性和一致性。

## 7. 通知与反馈
- **实时通知**：任务完成、漂移告警、门禁拦截通过顶部通知中心推送，点击跳转相关页面。
- **操作反馈**：所有操作（保存、删除、执行）均有Toast提示，耗时操作（如任务执行）显示进度条或加载状态。
- **错误处理**：API请求失败时，统一弹窗提示错误详情，并提供重试按钮。

## 8. 前端技术栈建议
- **框架**：React 18 + TypeScript
- **UI库**：Ant Design 5.x
- **状态管理**：Zustand / Redux Toolkit
- **图表**：ECharts / Ant Design Charts
- **实时通信**：Socket.IO-client
- **代码编辑器**：Monaco Editor
- **路由**：React Router v6
- **构建工具**：Vite

## 9. 后续优化方向
- **移动端适配**：为移动设备提供轻量版（仅查看功能）。
- **暗色模式**：系统级跟随+手动切换。
- **国际化**：支持中英文切换。
